Markdown Previewer

Objectives

  • When text is entered into the editor, the preview is updated.
  • Markdown text is entered in the editor and translated to HTML in the preview.
  • Default text in the editor has markdowns for the header, sub-header, link, inline code, code block, list item, blockquote, image, and bolded text.
  • Default markdowns in the editor are rendered as HTML when the previewer loads.

Background

First, I created the boiler plate code for this project. Inside the container div, I created two divs with the top and bottom class. These divs would hold the editor and preview elements. After I set up my code files and created the boiler plate code, I created the defaultMarkdown variable with a string of all the required markdowns for the project. The rest of the project was spent coding, testing, and debugging code to ensure each of the markdowns would appear in the previewer.

In order to get the markdowns to appear in the previewer, I created a construction method before the render method so I could create a temporary class component. Inside the constructor was super(props) and this.state object with the markdown property set to the defaultMarkdown variable. Inside the render method, I added an expression marked(this.state.markdown) so the defaultMarkdown string is printed out as text. In order to get the text to print as HTML, I added the dangerouslySetInnerHTML attribute of the preview id. I created an object with the __html property and set it with marked(this.state.markdown).

Once I got the required markdowns to work on the site, I added code to interpret carriage returns as
and open links in new tabs. In order to get carriage returns to work on the previewer, I used setOptions function with marked. Inside the object, I set the breaks to true. After reading the Marked.js documentation, I created a renderer variable before marked.setOptions which is set to new marked.Renderer(). Then I set renderer.link as a function so it would return links with the target attribute set to “_blank”.
