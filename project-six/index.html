Yelpcamp

Objectives

  • Create routes so users can create, edit, and delete campgrounds.
  • Set up user, comments, and campgrounds models to use inside routes.
  • Add authentication for contributing comments, logging into the site, and show specific elements on the page.
  • Debug, test, and refactor code.
  • Deploy web app on Heroku.

Background

At the beginning of this project, I kept the setup pretty simple by installing the packages I needed and getting the layout set up for the landing and campground pages. Once the dependencies were installed, I created the layout for the site and did some initial styling for the site using Bootstrap. Then I concentrated on how to initially get new campgrounds created. This resulted in two new routes added to the app.js file to post and show campgrounds on the site.

Now that a working version of the app was up and running, I started making several versions of this site where I added new routes to add and display campgrounds. I set up a schema and model for the campgrounds. Next, I created a show route and template to show information about the campgrounds and comments. I created a database to start collecting campground information and adding code to ensure the app could find and use this data in various models and templates.

A large amount of time was spent on authenticating the site. This involved creating a user model as well as setting up more routes that allow signed in users to edit, modify, and delete campgrounds or comments. The hardest part of this project was actually getting the web application to deploy on Heroku. I ran into a lot of database issues when I was trying to connect the web app to Heroku. It took some googling, but I discovered the url I was trying to use was incorrect. Once I fixed the url, my database was able to start collecting data with no error messages.
